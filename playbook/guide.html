AI Native Playbook

The step-by-step guide to transforming Benson into an AI-native team. Your tasks, your deliverables, your timeline.

Start January 13, 2026
End February 14, 2026
Duration 5 Weeks

The Goal

By February 14, 2026:

Phillip can describe a feature in plain English and ship it to production—with AI handling the implementation and the team providing judgment, taste, and quality assurance.

We're not replacing anyone. We're giving everyone superpowers. AI handles the repetitive work. Humans provide the judgment that makes Benson feel like Benson.

AI Does

  • Write code that matches our patterns
  • Generate screens from descriptions
  • Iterate based on feedback
  • Work without breaks

We Do

  • Decide what to build and why
  • Judge if designs feel right
  • Ensure code quality
  • Apply taste and judgment

How We Work Together

Every feature follows this flow. Everyone has a clear role. The order matters.

1

Phillip has an idea

Identifies a feature, improvement, or fix that users need

↓
2

Phillip pulls from iOS + Backend repos

Understands what exists: current screens, available data, API endpoints

↓
3

Phillip writes spec + generates with AI

Describes the feature in plain English. AI generates backend + iOS code and design variations.

↓
4

Luka reviews backend code

Checks API endpoints, database queries, business logic. Approves or requests changes. Backend must be solid before UI review.

↓
5

Tim & Tea review the designs

Look at AI-generated screens knowing what data is available. Give specific, actionable feedback. AI regenerates.

↓
6

Rok reviews iOS code

Final quality gate. Checks SwiftUI, components, animations, style compliance. Approves or requests changes.

↓
7

Ship to production

Merge, deploy, measure results. Loop back for improvements.

The order matters: Phillip initiates → Luka ensures backend is solid → Tim & Tea review designs knowing what data exists → Rok is the final code quality gate. AI does the heavy lifting. Everyone provides judgment.

Example Feedback Format

# Point System - Review Round 2
**Reviewer:** Tea
**Date:** Jan 15, 2026

---

## Variation A

### What works:
- Header layout is clean
- Green accent matches our brand

### What needs to change:
1. **Points number:** Too small. Increase by 50%.
2. **"Level 5" text:** Competes with points. Reduce opacity to 60%.
3. **Button:** "Earn More Points" is passive. Change to filled
   green button with "Start Earning"
4. **Progress bar:** Add percentage label (e.g., "72% to Level 6")

### What feels off (can't pinpoint):
- Hierarchy feels wrong - my eye doesn't know where to look first

### Verdict: 6/10

---

## Final recommendation:
Go with Variation C. Apply button fix from Variation A notes.

The 4 Buttons You Need

Button When to Use
Pull Start of any work session - get latest files
Commit After saving your feedback file - save locally
Push After commit - send to team
Discard If you mess up and want to undo changes
